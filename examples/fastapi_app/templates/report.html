Summary

The ferropdf project uses proven Rust libraries (html5ever, cssparser, taffy, pdf-writer) to convert HTML/CSS to PDF at high performance. The engine releases the Python GIL during rendering, making it perfectly compatible with async frameworks like FastAPI.

Architecture

Pipeline: HTML → DOM (html5ever) → Styles (cssparser) → Layout (taffy) → Pages (fragmentation) → PDF (pdf-writer). Each step is an isolated Rust crate with clear responsibilities.
